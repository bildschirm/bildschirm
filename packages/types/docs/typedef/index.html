    public SocketEvents: * source

    As ESDoc lacks a way to properly document events, this typedef shows all the different events the client might emit. The "type" below is the callback argument for the listener.

    Properties:

    NameTypeAttributeDescription
    connect void

    Emitted on a successful connect to the server. Called after authentication.
    disconnect DISCONNECT_REASON

    Emitted when the client disconnected from the server. The disconnect reason indicates why.
    error SOCKET_ERROR, object

    Emitted when the encounters an error. The first argument is the SOCKET_ERROR object, indicating what the error object might be.
    reconnecting Number

    Emitted once the clients starts trying to reconnect to the server. Attempt number passed to the listener.

    Example:

    client.on('error', (errorType, errorData) => {});
client.on('disconnect', reason => {});
client.on('reconnecting', attempt => {});
